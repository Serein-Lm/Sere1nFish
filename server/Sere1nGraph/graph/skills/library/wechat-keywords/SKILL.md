# 微信公众号搜索词库

将 `{company}` 替换为目标公司的规范名、品牌简称或控股单位名称。搜索结果应进入文章，复制原始链接后交给浏览器池读取 HTML、图片和上下文。

## 核心检索模板

| 场景 | 搜索词模板 | 重点信息 |
| --- | --- | --- |
| 招投标 | `{company} 招标`、`{company} 中标`、`{company} 采购` | 项目名称、采购人、供应商、联系人、联系方式 |
| 招商合作 | `{company} 招商`、`{company} 合作`、`{company} 入驻` | 合作条件、业务部门、联系方式 |
| 组织人员 | `{company} 任命`、`{company} 负责人`、`{company} 通讯录` | 姓名、职务、部门及上下文 |
| 招聘人才 | `{company} 招聘`、`{company} 校招`、`{company} 实习` | 部门、岗位、邮箱、电话、工作地点 |
| 公开联络 | `{company} 联系方式`、`{company} 联系我们` | 电话、邮箱、地址及用途 |
| 官方动态 | `{company} 公众号`、`{company} 新闻`、`{company} 公告` | 官方账号、关联单位、近期业务 |

## 组合规则

1. 根公司与第一层 100% 控股单位分别生成检索词，保留各自 `target_id` 和控股关系标注。
2. 优先使用品牌简称检索，同时保留企业规范全称用于确认归属。
3. 显式任务关键词优先，数据库词库用于补全，不覆盖用户输入。
4. 同一项目内按规范化空格去重；根公司和子单位的相同文章由原始链接去重。
5. 只把含有完整上下文的联系方式作为有效结果，避免脱离原文保存孤立号码。
