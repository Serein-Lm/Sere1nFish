# IT 支持场景案例

## 案例 1: 安全漏洞通知
**背景**: 目标为公司 IT 部门运维
**伪造身份**: 安全厂商技术支持
**逻辑链条**: 邮件告知安全漏洞 → 电话确认 → 发送"修复工具"
**关键点**: 利用安全焦虑心理，运维人员对漏洞修复有紧迫感
**结果**: 成功 — 目标在测试环境运行了"修复工具"

## 案例 2: 内部系统升级
**背景**: 已获取内网邮箱权限
**伪造身份**: IT 部门
**逻辑链条**: 内部邮件通知系统升级 → 附带"升级工具"下载链接
**关键点**: 内部邮件的信任度远高于外部
**结果**: 成功 — 多名员工下载运行了"升级工具"
