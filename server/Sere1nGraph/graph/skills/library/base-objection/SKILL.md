# 通用质疑应对 Skill

## objection 阶段

输出到 `ObjectionOutput.scenario_objections: list[ScenarioObjections]`。

### 输出规则

1. 确定当前有几个场景，**每个场景都必须**生成对应的 `ScenarioObjections`
2. 默认每个场景生成 **5 条** `ObjectionItem`
3. 每条 `ObjectionItem` 必须填充所有字段：

| 字段 | 规则 |
|------|------|
| `objection` | 目标可能提出的质疑（具体、口语化） |
| `response` | 应对话术（具体、可直接使用） |
| `tactic` | 使用的心理策略名称 |
| `context_note` | 与当前场景的具体关联说明（不能写泛泛的内容） |

### 常见质疑类型（必须覆盖）

1. 身份验证类："你是谁？我怎么没听说过？"
2. 流程质疑类："为什么不通过正式渠道？"
3. 安全警觉类："这个文件安全吗？"
4. 权限质疑类："你有权限处理这个吗？"
5. 时间质疑类："为什么这么急？"

### 心理策略库

| 策略 | 用法 | 示例 |
|------|------|------|
| 权威效应 | 引用上级/领导的指示 | "XX总直接交代的" |
| 互惠原则 | 强调已经为对方做了什么 | "方案我已经帮您整理好了" |
| 社会认同 | 提及其他人也这样做 | "您同事王工也是这样操作的" |
| 紧迫感 | 强调时间紧迫性 | "截止日期是明天" |
| 稀缺性 | 强调机会难得 | "这个名额只剩最后一个" |
| 一致性 | 利用对方之前的承诺 | "您之前说过会看的" |
| 好感原则 | 建立个人好感 | "久仰大名" |
| 虚荣心 | 满足对方的自尊需求 | "听说您是这方面的专家" |
