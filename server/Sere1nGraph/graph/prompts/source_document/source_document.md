# 目标证据判定

`subject_match` 衡量正文中是否存在可直接归属于目标实体的实质、可核验事实，不衡量公众号、作者或发布者是不是目标实体。目标不需要是整篇文章的唯一主体。

- 90-100：文章主要围绕目标，或者独立条目/章节明确写出目标名称，并提供项目、事件、产品、金额、时间、地点、联系人等多项具体事实。
- 80-89：独立条目/章节明确写出目标名称，并至少提供一项有采集价值的具体事实；行业招投标汇总中清晰完整的目标项目可进入此区间。
- 40-79：存在目标名称和一定上下文，但事实较少、归属关系不够明确或价值有限。
- 0-39：只偶然提及目标、仅为同地区或同行业、只有模糊关联，或者正文事实实际属于其他主体。

不得因为文章是行业汇总、第三方发布或同时包含多个主体就机械降分；也不得因为同属一个行业、关键词相似或仅出现目标名称就给高分。别名只有在输入明确标为可靠别名时才能作为目标身份依据。

# 价值评分

`relevance_score` 同时考虑目标证据强度和信息价值。直接反映目标采购、招标、业务需求、合作、人员、联系方式、系统或关键事件的内容应提高价值分；无具体事实的宣传、泛行业信息和偶然提及应降低价值分。

两个分数必须输出 0 到 100 的百分制数字，例如 `95`，禁止输出 0 到 1 的小数比例。`score_reason` 用一句话指出正文中的目标证据和评分依据。最终输出必须符合调用方提供的结构化 Schema。
