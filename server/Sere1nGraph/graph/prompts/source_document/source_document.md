# 目标证据判定

先判断 `article_scope`，再给 `subject_match`。`subject_match` 衡量目标实体在整篇文章主题、标题和有效篇幅中的聚焦程度，而不是判断正文是否曾经提到目标。

- `target_focused`：标题和主要篇幅围绕目标，或整篇核心事件明确以目标为主体。可按证据给 70-100 分。
- `multi_entity_roundup`：行业汇总、名单、资讯合集、招投标集合等同时包含多个互不从属主体，目标只是其中一个条目。即使该条目事实具体，`subject_match` 也不得超过 39。
- `incidental`：目标仅被顺带提及、出现在背景或与正文核心事件无关，`subject_match` 不得超过 20。
- `uncertain`：无法可靠判断文章主体，`subject_match` 不得达到高相关阈值。

第三方发布不等于低相关，只要整篇核心内容确实围绕目标仍可使用 `target_focused`。但不得把多主体汇总中的单条命中、同地区、同行业、关键词相似或仅出现目标名称评为高主体相关。别名只有在输入明确标为可靠别名时才能作为目标身份依据。

# 摘要与联系方式归属

- `summary` 只概括可直接归属于目标实体的事实，不得概括其他单位、行业列表或整篇无关内容；没有足够目标事实时留空。
- `target_contact_values` 只填写正文明确说明属于目标实体、目标部门或目标项目负责人的联系方式原值。
- 公众号发布方、协会页脚、作者、平台客服、其他条目或其他单位的电话、邮箱、微信、QQ 不属于目标，禁止写入 `target_contact_values`。
- 无法确认联系方式归属时留空，不得仅因联系方式出现在同一篇文章中就归给目标。

# 价值评分

`relevance_score` 考虑目标证据的信息价值，但不能替代 `subject_match`。多主体汇总中的单条信息可以有一定事实价值，仍不得通过高主体相关筛选。直接反映目标采购、招标、业务需求、合作、人员、联系方式、系统或关键事件的目标聚焦文章应提高价值分；无具体事实的宣传、泛行业信息和偶然提及应降低价值分。

两个分数必须输出 0 到 100 的百分制数字，例如 `95`，禁止输出 0 到 1 的小数比例。`score_reason` 用一句话指出正文中的目标证据和评分依据。最终输出必须符合调用方提供的结构化 Schema。
