# 2026-03-30 VibeCoding Log

## Docker Chrome 容器健康监控 + 热切换

### 改动文件
- `browser_manager/provider.py`
- `api/services/url_scan_pipeline.py`
- `Sere1nGraph/graph/agents/runtime.py`
- `config.json`

### 核心改动

1. **容器健康监控** — `_health_checker` 协程每 30s 通过 Docker stats API 查内存，超 1.5G 标记 unhealthy
2. **容器热切换** — `report_error` / `should_hot_swap` / `hot_swap_container`，连续 3 次容器级错误自动切换
3. **预热池** — `_warm_pool_filler` 维持 N 个空闲容器，热切换时秒级可用
4. **每个 URL 独占容器** — scan worker 每扫完一个 URL 就释放容器，彻底避免状态污染
5. **容器级错误快速中断** — `Network.enable timed out` / `403` / `Could not connect` 3 次就中断 agent，不浪费重试
6. **动态内存重启** — 低于 800M 不重启 Chrome，超过才重启

### 配置新增
```json
"memory_unhealthy_mb": 1500,
"memory_restart_mb": 800,
"health_check_interval": 30,
"warm_pool_size": 3,
"max_consecutive_errors": 3
```
