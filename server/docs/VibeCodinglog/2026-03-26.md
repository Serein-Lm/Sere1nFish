# VibeCoding Log — 2026-03-26

## 本次进度

### 1. Skill 系统重构（渐进式披露架构）

删除旧的 copywriting prompts 和 workflow，用渐进式披露三层架构重建：

- Layer 1 · Index — `SkillIndex` Pydantic 模型，Registry 只持有这一层
- Layer 2 · SKILL.md — YAML frontmatter + Markdown body，触发时加载
- Layer 3 · references/ — 按需加载的案例库

13 个内置 Skill，按渠道（wechat/email/phone/intranet/sms）和场景（recruitment/vendor/it-support）分类。Skill 通过 `phases` 列表实现跨阶段复用。每个 SKILL.md 明确引用 Pydantic Schema 字段名。

### 2. Pydantic Schema 结构化输出

所有输出统一为 Pydantic JSON Schema（`skills/schemas.py`）：ScenarioOutput、ScriptOutput、ObjectionOutput、FinalOutput、FindingCopywriting、UrlScanTask、InfoFinding 等。

### 3. URL 扫描 + 话术生成 Pipeline

`api/services/url_scan_pipeline.py` — 纯编排层，复用所有现有工具（normalize_url / probe_urls_batch / WebTaggingPipeline / extract_json_object）。

### 4. Browser 架构升级

playwright MCP 全部替换为 chrome-devtools MCP。4 个 agent 的 `server_name` 改为 `"chrome-devtools"`。Pipeline 运行时从 DockerProvider 动态获取容器 CDP 地址。

### 5. 观测层（Token Tracking）

`Sere1nGraph/graph/observability/` — 多层级 Token 观测：
- `pricing.py` — 模型定价（阶梯 + 固定）
- `tracker.py` — TokenTracker + LangChain Callback
- 一个 callback 自动注入到所有 `create_llm()` 创建的 LLM
- contextvars 上下文栈管理层级
- MongoDB 双写持久化（`token_usage_records` 集合）
- 逐层按需查询接口（不一次拉全量）

### 6. API 文档

- `docs/FRONTEND_API.md` — 前端完整对接文档
- `docs/UNIFIED_TASK_API.md` — 统一任务下发 API
- `docs/URL_SCAN_API.md` — URL 扫描数据模型
- `docs/OBSERVABILITY_API.md` — 观测层 API
- `docs/SKILL_SYSTEM_API.md` — Skill 系统 API

### 7. 统一任务下发接口

`api/routers/tasks.py` — 一个入口 `POST /api/v1/tasks/create`，通过 `task_type` 分发：
- `url_scan` → `UrlScanPipeline`
- `xhs_search` → `run_xhs_pipeline()`
- `douyin_search` → `run_douyin_pipeline()`
- `web_tagging` → `run_web_tagging_pipeline()`

`TASK_DISPATCHERS` 注册表模式，加新 pipeline 只需一行代码。

### 8. 话术生成改为 ReAct Agent

- `tools/skill_tools.py` — 3 个 LangChain tool（list_available_skills / load_skill / load_skill_reference）
- `prompts/copywriting/copywriting.md` — Step-by-step system prompt（Step1 分析 → Step2 加载skill → Step3 场景 → Step4 话术 → Step5 质疑 → Step6 输出）
- `factory.py` 新增 `create_copywriting_agent()` — ReAct agent，注入 skill tools
- Agent 自主决定加载哪些 skill，前端不需要选

### 9. 去掉 selected_categories + 三层上下文

- API 参数删掉 `selected_categories`，Skill 选择由 ReAct Agent 自主决定
- 传给 agent 的上下文三层：网站信息 + 当前 finding + 同 URL 其他 findings 摘要

### 10. 观测层 MongoDB 持久化 + 逐层接口

- `_record()` 双写：内存 + MongoDB（异步不阻塞）
- 查询内存优先，进程重启后从 MongoDB 聚合
- 逐层按需接口（前端点击逻辑）：
  - `GET /stats/global` — 全局汇总 + 项目列表（含 by_model + duration）
  - `GET /stats/project/{id}` — 项目汇总 + 任务列表（含 by_model + duration）
  - `GET /stats/task/{id}` — 任务汇总 + 各 Agent 明细
  - `GET /stats/records` — 原始调用记录
