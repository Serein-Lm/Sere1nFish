## 1. API 统一分页标准化

### 改动文件
- `api/schemas/pagination.py`（新增）
- `api/schemas/__init__.py`（新增）
- `api/routers/projects.py`
- `api/routers/project_api.py`

- `api/dao/web_tagging.py`
- `api/dao/findings.py`
- `api/services/company_scan_pipeline.py`

### 核心改动
- 所有列表查询 API 从 GET+query params 改为 POST+JSON Body
- 统一分页请求：`{ page: 1, page_size: 10 }`（默认每页 10 条）
- 统一分页响应：`{ items, total, page, page_size }`
- 新增 `PageRequest` 基类和各业务查询请求模型
- 所有 DAO 层 list 函数返回 `(items, total)` 元组，不再需要单独的 count 调用
- 删除了 douyin 的冗余 `/count` 端点（分页响应已自带 total）

### 路由命名
- 有冲突的用 `/list` 后缀：`POST /projects/list`、`POST /projects/{id}/tasks/list`
- 无冲突的直接复用原路径：`POST /projects/{id}/findings`、`POST /projects/{id}/notes` 等

## 4. 画像 ↔ Finding 双向关联

### 问题
画像表（xhs_profiles / douyin_profiles）没有 `finding_id`，前端无法从画像直接查话术。抖音 pipeline 完全没创建 findings。

### 改动文件
- `api/services/xhs_pipeline.py`
- `api/services/douyin_pipeline.py`
- `api/models/xhs_schema.py`
- `api/routers/xhs.py`
- `api/routers/douyin.py`
- `api/routers/project_api.py`
- `test_server/tests/test_backfill_finding_id.py`（新增）

### 核心改动
- XHS pipeline：创建 finding 后回写 `finding_id` 到 `xhs_profiles` 表
- 抖音 pipeline：新增 `_create_finding_for_profile` 方法，每个画像创建对应 finding + 回写 `finding_id` + 写入统一 profiles 表
- `XhsProfileOut` 和 `ProfileOut` 模型加 `finding_id` 字段
- `_profile_out` 输出函数加 `finding_id`
- 新增 3 个桥接 API（兼容历史数据）：
  - `GET /profiles/xhs/{user_id}/finding` — 优先画像表 finding_id，fallback findings 表反查
  - `GET /profiles/douyin/{sec_uid}/finding` — 同上
  - `GET /profiles/{user_id}/copywriting?source=xhs` — 一步到位查话术
- 新增 `test_backfill_finding_id.py` 脚本为历史数据补 `finding_id`

### 数据流
```
画像创建 → 创建 finding → finding_id 回写画像表
前端查话术：画像.finding_id → GET /findings/{fid}/copywriting
```

## 5. 截图 Docker 容器崩溃恢复

### 问题
`Page.evaluate: Target crashed` — 42 个用户连续截图，Chrome 渲染进程 OOM 崩溃。Docker 层有 `report_error` / `hot_swap_container` 机制但截图代码没调用。

### 改动文件
- `api/services/xhs_vision_tools.py`
- `crawler_tools/screenshot_tool.py`

### 核心改动
截图异常时不再只是 `release_cdp_endpoint`，改为：
1. `report_error` 上报错误
2. `should_hot_swap` 判断是否达到阈值
3. 达到阈值 → `hot_swap_container` 销毁旧容器 + 从预热池拿新容器
4. 重试时自动连到新的健康容器

## 7. 画像 Agent 重试机制

### 问题
`[XHS-画像] ✗ 解析失败` — Agent 返回内容但解析不出 JSON，直接跳过没有重试。

### 改动文件
- `api/services/xhs_pipeline.py`

## 8. Pipeline 话术 Worker 挂起修复

### 问题
Pipeline 跑完话术后卡住不结束。原因：`_prof_done` 是 `asyncio.Event()`，每个 profile worker 结束时都 `set()`，第一个 worker 完成就 set 了，话术 worker 看到 `_prof_done.is_set() + cw_q.empty()` 就退出，但其他 profile worker 还在往 `cw_q` 放数据。

### 改动文件
- `api/services/xhs_pipeline.py`

### 修复
- 从每个 `_prof_w` worker 里移除 `_prof_done.set()`
- 改为在 `await asyncio.gather(*pw)` 之后才 `_prof_done.set()`
- 确保所有 profile worker 都完成后话术 worker 才退出
