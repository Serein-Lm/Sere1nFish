## MCP + Docker Chrome 链路修复

- 定位 `unhandled errors in a TaskGroup` 根因：Apple Silicon 下 CDP 端口不通，chrome-devtools-mcp 的 `--browserUrl` 无法请求 `/json/version`
- 通过 deepwiki 发现 `--wsEndpoint` 参数，直接传 WebSocket 地址绕过 HTTP 发现
- `_get_chrome_cdp_url()` 改为返回 `ws://` 地址，`_override_chrome_mcp_config()` 用 `--wsEndpoint` 替换 `--browserUrl`
- 端到端测试验证：Docker 容器创建 → MCP 连接 → 29 个工具全部可用 → Agent 扫描成功

## Token 统计修复

- `create_agent_node` / `create_web_tagging_agent` 新增 `streaming` 参数透传到 `create_llm`
- 真实扫描用 `streaming=False`（后台任务不需要流式），DashScope 非流式模式返回完整 token 用量
- 验证：2 次 LLM 调用，35504 input + 397 output tokens，费用 ¥0.15

## 统一 RESTful API 重构

核心：所有资源嵌套在 project 下，findings 为核心数据模型。

新建文件：
- `api/dao/findings.py` — 统一 findings DAO（CRUD + summary 聚合 + 分页查询）
- `api/routers/project_api.py` — 嵌套 RESTful 路由

新增 MongoDB 集合：
- `findings` — 统一 findings（所有 source 汇聚）
- `copywritings` — 统一话术
- `profiles` — 统一画像

API 路由变更（旧 `/tasks/*` 已移除）：
- `POST /projects/{id}/tasks` — 下发任务
- `GET /projects/{id}/tasks` — 列出任务
- `GET /projects/{id}/findings/summary` — findings 看板统计
- `GET /projects/{id}/findings` — 分页查询（source/type/score/sort/limit/skip）
- `GET /findings/{fid}` — finding 详情
- `GET /findings/{fid}/profile` — 人物画像
- `GET /findings/{fid}/copywriting` — 话术
- `GET /findings/{fid}/notes` — 关联笔记

Pipeline 存储同步：
- `url_scan_pipeline` 的 findings 和 copywritings 同时写入统一集合（带 `source: "web_tagging"`）

## 任务删除 API

- `DELETE /projects/{id}/tasks/{tid}` — 删除任务及关联 findings/copywritings
- `DELETE /projects/{id}/tasks?status=error` — 批量删除
